Summary​


Project Akita: Implement and Build Docusaurus CMS Site

Join this squad to set up a contribution docs site to enable aspiring designers and devs to join open-source project workathons

Level of Difficulty: Beginner
Role: HTML and CSS Frontend Developer, Content Designer


### Proj Name Trivia

Proj Name Trivia​

This feature is named after the Akita dog.

The Akita is a muscular, double-coated dog of ancient Japanese lineage famous for their dignity, courage, and loyalty. In their native land, they're venerated as family protectors and symbols of good health, happiness, and long life.


Work Plan Overview​

Deliverable​

Develop a documentation site for Open Sprints, a collaborative workspace fro digital content editors, hackathon participants, and experienced mentors. (aspiring designers and developers)

Current State​

Pre-launch phase. The landing page for Open Sprints contains messaging that enables participation from volunteers (quick definition, roadmap, get involved). Open Sprints is a community hackathon event that has not launched yet. There are a few volunteers in UX and Dev

Goals and Mission​

Open Sprints mission is to teach early-career techies gain confidence in building within an agile-driven environment. 


Proposal​

Open Sprints landing page may have gaps in the experience. Ensure that the experience has a way for volunteer and mentors to sign up and so that partners also understand the overall mission and vision of both the community event and the long term goals of the org.

Ultimately, our call to action is to get sign ups who would be

1) enable application for workathon mentors

2) be interested in volunteering to maintain the site (Markdown content editors, PR Reviews, General Maintenance)

3) be interested in becoming participants in the monthly in person workathon.

Constraints​

1) Dynamic data or Third-party API embedding. The system should be able to string the Airtable Form into the landing page

2) Think ahead for sustainable solutions and downstream maintenance. For example, the contributor page may have to be built manually now, but can be refined later to use Airtable gallery view embedding on a page

3) Tech team should be able to plan maturity path and phase what's MVP and what's beyond scope for these experiences.
